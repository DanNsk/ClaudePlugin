# Question Bank

Reference prompts for brainstorming. Use as inspiration, not a checklist.

## Understanding the Problem
- What problem are we actually solving?
- What triggered this need now?
- What happens today without this?
- What's the cost of NOT solving this?
- What would success look like?

## Stakeholders
- Who uses this directly?
- Who's affected indirectly?
- Who approves / pays for this?
- Who might resist this change?

## Challenging Assumptions
- What if [stated assumption] isn't true?
- How confident are we about [fact]?
- Is [constraint] actually a constraint?
- What's the minimum viable version?

## Edge Cases
- What happens when [resource] is unavailable?
- What about [unusual user type]?
- What if [input] is malformed/missing/huge?
- What about [timezone/locale/currency]?

## Scope
- What's explicitly OUT of scope?
- Where does this feature's responsibility end?
- What adjacent problems should we NOT solve?

## Constraints
- Timeline pressures?
- Budget limits?
- Compliance/regulatory requirements?
- Dependencies on other teams?
- Technical limitations?

## Prior Art
- Has this been attempted before?
- Similar features in competing products?
- What can we learn from failed attempts?
